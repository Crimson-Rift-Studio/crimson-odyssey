# Crimson Odyssey v0.3.0 Finishing Plan

## Product goal

Make first-run setup and future upgrades simple enough that a user can operate Crimson Odyssey without manually editing runtime configuration files.

## Delivery plan

### Phase 1: Onboarding foundation

- Full setup wizard
- Identity and Soul choices
- Secure provider credential flow
- Provider-first model selection
- Starter Loadout profiles

### Phase 2: Operational setup

- Security profile selection
- History retention selection
- Telegram and Discord onboarding
- Doctor summary and optional TUI launch

### Phase 3: Update lifecycle

- Cached update checks
- Git and global npm installation detection
- Notification, ask, auto, and disabled modes
- Dirty-worktree protection
- Post-update source verification
- Interactive startup update notice

### Phase 4: Graduation verification

- Syntax verification
- Full automated test suite
- Secret scan
- No-em-dash policy
- Package dry-run
- Local package installation
- CLI version, setup, doctor, and update smoke tests
- TUI layout matrix
- Cross-platform GitHub Actions

## Definition of finished

Version 0.3.0 is finished when a new user can run `crimson setup` without touching `.env`, all non-secret configuration is written automatically, credentials are stored by reference only, update behavior is configurable, update notices appear during interactive startup and through CLI commands, auto update refuses to overwrite local changes, workspace state survives updates, and every automated verification gate passes.
